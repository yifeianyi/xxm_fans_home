## 1. 开发环境配置

开发环境只需修改后端配置，完成数据库迁移即可测试。

### 修改歌手配置

编辑 `repo/xxm_fans_backend/songlist/models.py`，添加歌手配置：

```python
# songlist/models.py
ARTIST_CONFIG = {
    'youyou': '乐游',
    'bingjie': '冰洁',
    'newartist': '新歌手',  # 添加这一行
}
```

### 创建并执行数据库迁移

```bash
cd repo/xxm_fans_backend

# 激活虚拟环境
source venv/bin/activate

# 创建迁移文件
python manage.py makemigrations songlist

# 执行迁移
python manage.py migrate songlist --database=songlist_db
```

**系统自动创建：**
- `NewartistSong` / `NewartistSiteSetting` 模型类
- `songlist_newartistsong` / `songlist_newartistsitesetting` 数据库表
- Admin 后台模块（动态注册）
- 所有 API 接口

**验证开发环境：**

```bash
# 启动开发服务器
python manage.py runserver

# 测试 API
curl "http://localhost:8000/api/songlist/site-settings/?artist=newartist"
```

> 💡 **提示**：开发环境使用 URL 参数访问不同歌手，如 `http://localhost:8080/songlist/?artist=newartist`，无需配置前端 `.env`。

## 2. 生产环境部署

> 💡 **自动化部署**：可以使用 `deploy_new_artist.sh` 脚本一键完成部署（推荐）

### 方式一：使用自动化脚本（推荐）

#### 前置条件

1. **本地提交并推送代码**

```bash
cd repo/xxm_fans_backend
git add songlist/models.py songlist/migrations/
git commit -m "feat(songlist): 添加新歌手 newartist"
git push origin main
```

2. **配置 DNS**（见第 4 节）

#### 执行部署脚本

```bash
# SSH 登录服务器
cd /home/yifeianyi/Desktop/xxm_fans_home/scripts

# 执行部署脚本
sudo ./deploy_new_artist.sh newartist newartist
```

脚本会自动完成：
1. 拉取后端代码更新
2. 执行数据库迁移
3. 配置前端域名映射（.env）
4. 配置 Nginx + SSL
5. 更新 ALLOWED_HOSTS
6. 重启后端服务

---

### 2.4 配置 Nginx + SSL

#### 方式一：使用部署脚本（推荐）

```bash
cd /home/yifeianyi/Desktop/xxm_fans_home/scripts
sudo ./setup_songlist_subdomain.sh newartist newartist
```

脚本会自动完成：
1. 创建 Nginx 配置文件
2. 启用配置并测试
3. 申请 SSL 证书
4. 更新前端域名配置（如果 `.env` 存在）
5. 重新构建前端

#### 方式二：手动配置

##### 步骤 1：创建 HTTP 配置（用于 Certbot 验证）

```bash
# 创建临时 HTTP 配置文件
sudo tee /etc/nginx/sites-available/newartist-songlist.conf > /dev/null << 'EOF'
# HTTP 服务器 - newartist.xxm8777.cn
server {
    listen 80;
    server_name newartist.xxm8777.cn;

    # 放行 Certbot 的 challenge
    location ^~ /.well-known/acme-challenge/ {
        root /home/yifeianyi/Desktop/xxm_fans_home/repo/TempSongListFrontend/dist;
        default_type "text/plain";
    }

    # 临时返回 200 以便 Certbot 验证
    location / {
        root /home/yifeianyi/Desktop/xxm_fans_home/repo/TempSongListFrontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 媒体文件
    location /media/ {
        alias /home/yifeianyi/Desktop/xxm_fans_home/media/;
        expires 30d;
    }
}
EOF

# 启用配置
sudo ln -sf /etc/nginx/sites-available/newartist-songlist.conf \
        /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

##### 步骤 2：申请 SSL 证书

```bash
sudo certbot --nginx -d newartist.xxm8777.cn --non-interactive --agree-tos --redirect
```

> ⚠️ **注意**：申请 SSL 证书前，确保域名 DNS 已解析到服务器。

---

### 2.5 更新 ALLOWED_HOSTS

编辑后端 `.env` 文件，添加新域名：

```bash
# 编辑环境变量文件
sudo nano /home/yifeianyi/Desktop/xxm_fans_home/repo/xxm_fans_backend/.env
```

添加新域名到 `DJANGO_ALLOWED_HOSTS`：

```bash
# 原配置
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1,47.122.126.246,xxm8777.cn,www.xxm8777.cn

# 更新后（添加 newartist.xxm8777.cn）
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1,47.122.126.246,xxm8777.cn,www.xxm8777.cn,newartist.xxm8777.cn
```

---

### 2.6 重启服务

```bash

sudo systemctl restart xxm-backend.service

# 重载 Nginx（加载新配置）
sudo systemctl reload nginx
```

## 4. 配置 DNS

在域名服务商处添加 A 记录：

| 类型 | 主机记录 | 记录值 | TTL |
|------|---------|--------|-----|
| A | newartist | 服务器IP | 600 |

获取服务器 IP：

```bash
curl ifconfig.me
```

> ⏱️ DNS 生效通常需要 10-30 分钟。

## 5. 常见问题

### Q1: API 返回 400 Bad Request

**原因**：域名未添加到 `ALLOWED_HOSTS`

**解决**：
```bash
# 编辑 .env 文件，添加域名
DJANGO_ALLOWED_HOSTS=...,newartist.xxm8777.cn

# 重启服务
sudo systemctl restart xxm-backend.service
```

### Q2: SSL 证书申请失败

**原因**：域名 DNS 未解析或配置错误

**解决**：
1. 检查 DNS 解析是否生效：`nslookup newartist.xxm8777.cn`
2. 确保域名指向正确的服务器 IP
3. 等待 DNS 生效后再申请证书

### Q3: Admin 后台不显示新歌手

**原因**：后端服务未重启，新代码未加载

**解决**：
```bash
sudo systemctl restart xxm-backend.service
```

### Q4: 图片无法显示

**原因**：`photo_url` 存储为 URL 编码格式

**解决**：已在代码中修复，确保使用的是最新代码版本。
