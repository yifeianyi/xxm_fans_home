# yukari.top/fans 功能分析文档

> 分析日期：2026-05-07
> 目标站点：https://yukari.top/fans
> 站点定位：小缘（Yukari）粉丝俱乐部网站

---

## 一、站点概述

yukari.top 是小缘（Yukari，B站知名唱见）的粉丝俱乐部网站，采用 Next.js 构建，提供直播日历、歌曲库管理、粉丝排行榜等核心功能。网站整体视觉风格为毛玻璃卡片式设计（glass-card），配合可爱的动漫图标导航。

**技术栈初步判断**：Next.js SSR、自定义数据集、B站 API 数据聚合。

---

## 二、核心功能模块

### 2.1 缘历（直播日历） — `/calendar`

**功能描述**：
- 月视图直播日历，以日历网格展示
- 有直播的日期高亮显示，点击可跳转到单场直播详情页（如 `/calendar/2026/05/01`）
- 支持前后月份切换导航
- 直播场次以日期数字下的链接标记呈现

**关键行为**：
- 当天日期有视觉区分
- 日期链接内嵌箭头/指示符标识

---

### 2.2 歌库（歌曲管理） — `/song`

**功能描述**：
- 歌曲数据库，以表格形式展示，支持搜索和分页（当前约 136 页）
- 每行显示：歌名 → 原唱 → 上次演唱日期 → 演唱次数
- 三个子页面：
  - **歌库**（`/song`）：全部歌曲列表，带搜索筛选
  - **投稿**（`/vdo`）：B站视频投稿列表
  - **歌单**（`/song/list`）：自定义歌曲播放列表

**关键行为**：
- 歌名点击跳转到单曲详情页（如 `/song/1298`）
- 分页控件支持跳转到指定页码
- 按"上次演唱"倒序排列（最近演唱的在前）

---

### 2.3 小呆毛（粉丝专区） — `/fans`

**功能概述**：粉丝专区是 yukari.top 最具特色的模块，包含三个子功能。

#### 2.3.1 出勤率排名 — `/fans`

**排名机制说明**：
- **出勤定义**：上舰/续舰、SC醒目留言、投喂礼物、发弹幕、观看一定时长，达成**任意一项**即视为成功出勤
- **结算时机**：每场直播结束后结算，短暂中断/重启视为同一场直播（以缘历为准）
- **更新时间**：每天凌晨 4:00
- **排名规则**：同一排名允许多人并列

**排名展示要求**：
| 排名类型 | 门槛条件 |
|---------|---------|
| 综合排名 | 首次出现的出勤次数 > 总直播次数的 40% |
| 年度排名 | 出勤次数 > 全年直播次数的 40% |

**页面展示**：
- 按出勤率百分比降序排列（100% → ... → 71.88%）
- 每行展示：排名数字 + 多名粉丝（头像 + B站昵称，可点击跳转 B站空间）
- 每级排名下有多人并列时，默认显示前几名，其余"点击显示 xx 位小呆毛"展开
- 无头像的粉丝显示默认占位图（`fans_64.png`）
- 分页支持

#### 2.3.2 弹幕排名 — `/fans/danmu`

**排名机制说明**：
- 只统计直播期间成功发送的弹幕（被屏蔽/未接收的不计入）
- 与出勤率排名不同：**同一排名只会有一个人**
- 门槛条件：弹幕量 > 400（暂定，之后改为总弹幕量的一定百分比）
- 更新时间：每天凌晨 4:30

**页面展示**：
- 按弹幕数量降序排列（约 29 页）
- 每行展示：排名 + 头像 + B站昵称 + 弹幕数量 + 占全年弹幕百分比
- 卡片式布局，头像 + 用户名大字号突出显示
- 点击跳转对应 B站用户空间

#### 2.3.3 搜索小呆毛 — `/fans/search`

**功能描述**：
- 粉丝搜索功能
- 数据每天凌晨更新
- 不符合要求或数据不足的粉丝显示为"未统计"

---

### 2.4 导航栏设计

- 左上角 Logo + 站点名（小缘粉丝俱乐部）
- 三个主导航入口使用图片图标：
  - 缘历（Calendar 图标）
  - 歌曲（音符图标）
  - 小呆毛（粉丝图标，当前页高亮）
- 每个导航项图标 + 文字双排展示
- 当前激活页有视觉高亮

### 2.5 页面底部（Footer）

- 站点 Logo + 名称
- 版权声明：`© 小缘粉丝俱乐部 YUKARI FAN CLUB`

## 四、可借鉴功能建议

### 4.1 粉丝排名系统（优先级：高）
yukari.top 最核心的差异化功能。XXM 可考虑：
- 接入 B站直播间数据（弹幕、礼物、SC、舰长等）
- 实现出勤率计算和排行榜
- 提供弹幕排名（按数量 + 占比）
- 从"粉丝站"升级为"粉丝俱乐部"，增强用户粘性

### 4.2 粉丝搜索（优先级：中）
- 按 B站用户名或 UID 搜索粉丝
- 展示粉丝的详细参与数据

### 4.3 歌曲详情页（优先级：中）
- yukari.top 的歌名可点击进入独立详情页
- 展示该歌曲的完整演唱历史、每次直播的时间和 BV号

### 4.4 视频投稿子页（优先级：低）
- 将 B站视频投稿独立为 `/vdo` 页面
- 与歌曲库形成区分：歌库展示"歌曲作品"，投稿展示"视频内容"

### 4.5 品牌文字标识（优先级：低）
- yukari.top 导航 Logo 旁有站点名称（小缘粉丝俱乐部）
- XXM 目前只有头像图标，可添加"满"或"小满虫之家"文字标识增强品牌识别

## 五、技术实现参考

### 5.1 出勤率计算逻辑（推测）
```
对于每场直播：
  IF 用户在直播期间：
    - 发送了弹幕 → 计入出勤
    - 投喂了礼物 → 计入出勤
    - 发送了 SC → 计入出勤
    - 上舰/续舰 → 计入出勤
    - 观看 ≥ N 分钟 → 计入出勤
  THEN 出勤次数 += 1

综合出勤率 = 出勤次数 / 统计期总直播次数 * 100%
```

### 5.2 数据更新策略
- 出勤率：每天凌晨 4:00 更新
- 弹幕排名：每天凌晨 4:30 更新
- 搜索数据：每天凌晨更新
- 采用夜间离线批处理而非实时计算

### 5.3 头像加载策略
- 有头像的用户：从 CDN 加载（`i.xiaoyuan.club/fans/avatar/{uid}`）
- 无头像的用户：显示默认占位图（`/fans_64.png`）
- 使用 Next.js `next/image` 优化加载
